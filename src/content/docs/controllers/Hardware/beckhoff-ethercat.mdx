Beckhoff EtherCAT terminals are a modular I/O system that works very well for LinuxCNC machines when you want industrial wiring, distributed I/O, and deterministic cycle timing.

In a LinuxCNC setup, the common architecture is:

1. LinuxCNC + EtherCAT master on the control PC.
2. Beckhoff coupler (usually `EK1100`) as EtherCAT head.
3. EL terminals for digital I/O, analog I/O, encoders, and motion.

## Why use Beckhoff terminals

- Robust DIN-rail hardware with clear electrical specs
- Easy expansion by adding terminals to the right of the coupler
- Good fit for mixed machines (limits, probes, spindle control, VFD I/O, toolchanger I/O)
- Strong option for distributed control cabinets

## LinuxCNC compatibility scope

Compatibility in practice is split into two groups:

1. Devices with dedicated support entries in `linuxcnc-ethercat` (`lcec`) device definitions.
2. Additional EtherCAT/CiA402 devices that can work via custom PDO mapping.

## Practical selection guide

For most LinuxCNC retrofit/new-build machines, a practical Beckhoff stack often starts like this:

1. `EK1100` coupler
2. `EL1008` or `EL1809` for machine inputs
3. `EL2008` or `EL2809` for outputs
4. `EL5101`/`EL5151` for encoder feedback (if needed)
5. `EL4002`/`EL4102` for analog command outputs (e.g. spindle/VFD)
6. `EL7031`/`EL7041` only if you want integrated terminal-level stepper drive

## Notes and caveats

- Always verify the exact order code (for example `-0010`, `-0052`, `-9415`) because electrical capability can differ from the base model.
- For motion terminals (stepper/servo), check cycle time and PDO mapping in your final LinuxCNC config before buying hardware in volume.
- Safety terminals (`EL69xx`) are not a shortcut for machine safety compliance; they require proper safety engineering.
